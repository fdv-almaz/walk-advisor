# Исправление: Правильное использование GPS координат при переключении режима

**Дата**: 2026-05-22
**Версия**: 2.0.2

---

## 📋 Проблема

При переключении с IP режима на GPS режим и нажатии кнопки "Обновить", приложение продолжало отображать старые данные, полученные по сети (IP), вместо свежих данных с GPS координат.

**Пример проблемы**:
1. Пользователь находится в режиме IP (например, Москва по IP)
2. Переключается на GPS (реально в другом городе, например, Санкт-Петербург)
3. Нажимает "Обновить"
4. Но приложение все еще показывает погоду для Москвы (старые IP данные)

---

## ✅ Решение

### 1. **При переключении режима на GPS - запрос свежих координат**

**Улучшение в `onReceive`**:
```swift
.onReceive(locationManager.$useIPLocation) { useIP in
    Task {
        if !useIP && locationManager.isAuthorized {
            // Переключились на GPS - запросим свежие координаты
            locationManager.requestLocation()
            // Небольшая задержка чтобы GPS обновился
            try? await Task.sleep(nanoseconds: 500_000_000)  // 0.5 секунды
        }
        await loadWeather()
    }
}
```

**Эффект**:
- ✅ При переключении на GPS активируется запрос GPS координат
- ✅ Даётся время GPS обновиться (500ms)
- ✅ Затем загружаются новые данные погоды

---

### 2. **При нажатии кнопки "Обновить" в GPS режиме - запрос свежих GPS координат**

**Улучшение в методе `loadWeather()`**:
```swift
if locationManager.useIPLocation {
    // IP режим: используем существующие IP координаты
} else {
    // GPS режим: запросим свежие GPS координаты
    if locationManager.isAuthorized {
        locationManager.requestLocation()
        // Ждем получения GPS координат (до 2 секунд)
        var gpsLocation: CLLocationCoordinate2D?
        for _ in 0..<20 {
            if let location = locationManager.location {
                gpsLocation = location
                break
            }
            try? await Task.sleep(nanoseconds: 100_000_000)  // 0.1 секунды
        }

        guard let location = gpsLocation else {
            errorMessage = localization.localize("no_location")
            return
        }
        latitude = location.latitude
        longitude = location.longitude
    }
}
```

**Логика работы**:
1. Запросить новые GPS координаты через `requestLocation()`
2. Ждать их получения с timeout до 2 секунд (20 итераций × 0.1s)
3. Использовать полученные свежие координаты для загрузки погоды
4. Если координаты не получены - показать ошибку

## 📊 Технические детали

### Timeout логика
- **При переключении**: 500ms на обновление GPS
- **При нажатии "Обновить"**: до 2 секунд (20 × 100ms)
- Позволяет GPS получить координаты, но не зависает надолго

### Обработка ошибок
- Если GPS не авторизирована → показываем prompt на IP
- Если GPS координаты не получены за 2 сек → ошибка "no_location"
- IP режим использует уже загруженные координаты (быстро)
